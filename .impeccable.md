# Design Context — Periodic Table Explorer

## Design Context

### Users
High school and university students studying chemistry. They use this as a reference and exploration tool — looking up elements, comparing properties, and discovering patterns across the periodic table. The context is both study sessions and casual curiosity. The interface should feel engaging enough to encourage exploration, not just lookup.

### Brand Personality
**Three words**: Scientific · Immersive · Luminous

The tone is futuristic and wonder-inducing — like navigating a starmap or interfacing with a sci-fi lab terminal. It should feel like the periodic table came alive. Students should feel curious and excited, not like they're reading a textbook.

### Aesthetic Direction
- **Theme**: Dark-mode only (deep navy/slate backgrounds already established)
- **Accents**: Cyan primary accent with category-specific element colors (already defined as CSS tokens)
- **Feel**: Futuristic lab terminal — precise data density combined with glowing, luminous details
- **Typography**: Should feel scientific but not clinical — readable at small sizes (element tiles are tiny), with a distinctive display font for headings/labels
- **Motion**: Purposeful — glow pulses, smooth state transitions, staggered reveals. GSAP is available.
- **Anti-references**: Avoid generic dark dashboard clichés — no glassmorphism for its own sake, no purple-to-blue gradients, no hero metric layouts
- **No strong external reference** — this should be distinctly its own thing. Make it memorable.

### Design Principles
1. **Data is the hero** — every design decision serves legibility and discoverability of element data, not decoration
2. **Luminous precision** — glows and color are earned through meaning (category colors), not applied decoratively
3. **Rhythm over uniformity** — spacing should breathe: tight within element tiles, generous between sections
4. **Wonder through density** — 118 elements packed into a grid should feel like a constellation, not a spreadsheet
5. **Student-first clarity** — hierarchy must be immediately readable; a student mid-study shouldn't have to think about the UI
